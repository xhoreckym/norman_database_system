Introduction

Base URL: http://127.0.0.1:8000

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer 1|your_token_here".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

How to get your token:

  1. Log in to the NORMAN Database System
  2. Go to your Dashboard → API Tokens
  3. Click Generate New Token
  4. Copy the token (it will only be shown once!)

Your token looks like this:

1|uQgwNnXHx7QJom31K8dfsdfsd8f7sd98f7sdf

"Try It Out" - What to paste in the Authorization field:

Copy your full token and paste it in the Authorization box:

1|uQgwNnXHx7QJom31K8dfsdfsd8f7sd98f7sdf

(Scribe automatically adds "Bearer " prefix)

Code Examples

curl:

curl -H "Authorization: Bearer 1|uQgwNnXHx7QJom31..." \
     "https://www.norman-network.com/nds/api/v1/empodat/country/SK"

Python:

import requests

headers = {"Authorization": "Bearer 1|uQgwNnXHx7QJom31..."}
response = requests.get(
    "https://www.norman-network.com/nds/api/v1/empodat/country/SK",
    headers=headers
)

JavaScript:

fetch("https://www.norman-network.com/nds/api/v1/empodat/country/SK", {
    headers: { "Authorization": "Bearer 1|uQgwNnXHx7QJom31..." }
})

EMPODAT

APIs for retrieving environmental monitoring data from the EMPODAT database. All endpoints require authentication via Sanctum token.

Search EMPODAT by substance or country

requires authentication

Retrieve EMPODAT records by searching for a substance (NORMAN ID or InChIKey) or by country code. Results are paginated due to potentially large datasets.

Example request:
curl --request GET \
    --get "http://127.0.0.1:8000/api/v1/empodat/substance/NS00000214?page=1&per_page=100" \
    --header "Authorization: Bearer 1|your_token_here" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://127.0.0.1:8000/api/v1/empodat/substance/NS00000214"
);

const params = {
    "page": "1",
    "per_page": "100",
};
Object.keys(params)
    .forEach(key => url.searchParams.append(key, params[key]));

const headers = {
    "Authorization": "Bearer 1|your_token_here",
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "GET",
    headers,
}).then(response => response.json());
$client = new \GuzzleHttp\Client();
$url = 'http://127.0.0.1:8000/api/v1/empodat/substance/NS00000214';
$response = $client->get(
    $url,
    [
        'headers' => [
            'Authorization' => 'Bearer 1|your_token_here',
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
        ],
        'query' => [
            'page' => '1',
            'per_page' => '100',
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));

Request      

GET api/v1/empodat/{search_type}/{search_value}

Headers

Authorization      

Example: Bearer 1|your_token_here

Content-Type      

Example: application/json

Accept      

Example: application/json

URL Parameters

search_type   string   

The type of search: "substance" or "country". Example: substance

search_value   string   

The search value. For substance: NS code (NS00000214) or InChIKey. For country: 2-letter ISO code (SK, DE). Example: NS00000214

Query Parameters

page   integer  optional  

The page number for pagination. Example: 1

per_page   integer  optional  

Number of records per page (max 1000, default 100). Example: 100

Substances

APIs for retrieving substance data from the NORMAN Suspect List Exchange database.

Get substance by NORMAN ID

Retrieve a substance using its NORMAN identifier (e.g., NS00004453). Returns comprehensive substance data including chemical identifiers, molecular properties, categories, and sources.

Example request:
curl --request GET \
    --get "http://127.0.0.1:8000/api/v1/substances/code/NS00004453" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"code\": \"NS75491716\"
}"

const url = new URL(
    "http://127.0.0.1:8000/api/v1/substances/code/NS00004453"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "code": "NS75491716"
};

fetch(url, {
    method: "GET",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());
$client = new \GuzzleHttp\Client();
$url = 'http://127.0.0.1:8000/api/v1/substances/code/NS00004453';
$response = $client->get(
    $url,
    [
        'headers' => [
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
        ],
        'json' => [
            'code' => 'NS75491716',
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));
